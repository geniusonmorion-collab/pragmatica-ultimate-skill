# Ресерч и вгруз

*An adaptive research system for product designers: a living project board, a dispatcher-copilot, a library of small research moves, and a two-track ideation engine.*

## Что это

Адаптивная система исследований для продуктовых дизайнеров. Пять частей:

- **Доска проекта** — живая память всех 9 этапов: карта бизнеса клиента, ставки с метками «знаю / предполагаю / не знаю», нестыковки, план проверок, морозильник идей и бюджет в часах дизайнера.
- **Напарник-диспетчер** — на любой застрявший вопрос предлагает меню из 2–3 маленьких исследований с ценой и эффектом, плюс законную опцию «не исследовать: принять как ставку и проверить дешёвым тестом позже».
- **Библиотека ходов** — ~58 маленьких исследований (час-два, не недели): от лестницы «зачем» и прогона руками до раскопки отзывов и разбора провала теста; с v2.2 — ветки «Цифры клиента» (воронка под решение, чтение чужих экспериментов, салфетка «сколько это стоит в метрике»…) и «Визуал» (коды категории, мудборд как ставка, 5-секундный тест эстетики…); с v2.3 — ветка «Генератор проб» (пробы-носители — серые HTML-протики, фейк-двери, варианты-безбилетники — делает агент под ставку с порогом, дизайнеру остаётся десятиминутка и вето) и Mobbin как источник визуального корпуса через официальный MCP; с v2.4 — B2B-ходы «Карта закупки», «Рычаги роста» и «Дерево владения метриками» плюс правила подачи и посадки (несущая ставка первым слайдом, посадка идей на поверхность, реверс без рук).
- **Двухконтурный движок идей** — Контур A поднимает пол (квота дерзости: смелая идея с проверкой в каждом проекте), Контур B охотится за прорывом (добыча нового сырья ходами Х1–Х6, сессия «Прорыв», проверка на козырь, дешёвые радикальные пробы).
- **Мультиплеер** (раздел 7) — протокол команды на одной доске: роли и права, метки без усреднения, райтер и аналитик пишут сами, конфликт решает тест, не должность.

## Для кого и чем отличается

Для продуктовых дизайнеров и дизайн-команд, которым исследование нужно не «этапом в начале», а по делу.

- **Исследование — сервис под застрявшее решение, а не этап.** Нет решения с именем — нет исследования. Вопросы возникают на вайрфреймах, контенте и тестах — система отвечает там, где они возникли.
- **Метрика — число изменённых решений**, а не объём текста. Ресерч, убивший половину брифа, — успех.
- **Колонка «не делать»** — вычёркивание встроено в каждый гейт: убрать шаг часто сильнее, чем проверить его.
- **Язык дизайнера снаружи, методология внутри.** Команда учит ровно 9 слов глоссария (ставка, зацепка, козырь, Aha-момент…) — остальное система переводит сама и никогда не поправляет язык дизайнера.

## Структура репозитория

```
research-vgruz/
├── README.md
├── METHODOLOGY.md                        # методология целиком, v2.4
├── BOARD.md                              # живая доска самой методологии:
│                                         # ставки, тесты, нестыковки — «едим
│                                         # свой корм»
├── skills/
│   └── pragmatica/                       # Claude-скилл — носитель методологии:
│       ├── SKILL.md                      # диспетчер (ядро)
│       └── references/                   # ходы, контракт доски, движок идей,
│                                         # линтер, языковой слой
└── design-notes/
    ├── ideation-engine-evolution.md      # происхождение Контура B: анатомия
    │                                     # 15 прорывов, ретро-тест, повторный
    │                                     # прогон «Долями»
    └── blind-run-contour-b.md            # слепые прогоны Контура B, попытки
                                          # №1–2: дизайн двойников, судейство,
                                          # оба брака, пересмотр инструмента
```

Папки `probes/` (протики-носители, раздел 4.3) и `research/visual-corpus/` (корпус экранов из Mobbin, раздел 4.2) — папки **рабочего клиентского проекта**, их создаёт скилл по ходу работы; в этом репозитории методологии их нет и не будет.

## Установка скилла

```bash
git clone https://github.com/geniusonmorion-collab/pragmatica-ultimate-skill.git
cp -r pragmatica-ultimate-skill/skills/pragmatica ~/.claude/skills/       # для всех проектов
# или в конкретный проект: cp -r … <проект>/.claude/skills/
```

Дальше в Claude Code: принесите бриф или застрявший вопрос — скилл `pragmatica` включится сам (или явно: `/pragmatica`). Доску он создаст в папке `research/` вашего рабочего проекта. Скилл прошёл дым-тест (чистый агент по одним файлам скилла собрал доску, атаковал цель и выдал меню с опцией «не исследовать»), но живых пилотов ещё не проходил — см. статус ниже.

## Статус

**v2.4** (03.07.2026): v1 — конвейер → v2 — адаптивная система → v2.1 — два контура движка идей → v2.2 — ветки «Цифры клиента» и «Визуал», мультиплеер, слепой прогон Контура B, доска методологии → v2.3 — ветка «Генератор проб» (носители делает агент) и Mobbin-интеграция (официальный MCP) → v2.4 — три B2B-хода («Карта закупки», «Рычаги роста», «Дерево владения метриками») и правила подачи и посадки: несущая ставка первым слайдом, посадка идей на поверхность, реверс без рук, профили несущих ходов, карта соответствия.

Честно: методология проверена **кабинетным пилотом на реальном кейсе** («Долями»: точка зрения, сессия идей, повторный прогон Контура B) и ретро-тестами на известных прорывах — ретро-счёт 2/2/1 добыт агентом, знавшим ответы (hindsight). **Слепой прогон Контура B дважды не состоялся:** попытка №1 забракована по гигиене процедуры (ключ судьи утёк прогоняющим, 0 засчитанных из 3), попытка №2 — контролем слепоты на pre-flight (все 4 брифа-двойника прозрачны для LLM-прогоняющего, 0 засчитанных из 4). Ставка не проверена ни в какую сторону; подтверждён pre-flight как дешёвый детектор, опровергнута достижимость слепоты маскировкой на реальных прототипах — дальше пересмотр инструмента или перенос веса на живой пилот; отчёты в `design-notes/blind-run-contour-b.md`. **Полуживой прогон на воркшопе пройден, фидбек вплавлен:** разбор с лидом вскрыл три системных пробела (B2B-цикл сделки, посадка идей на поверхность, подача без несущей ставки) — они стали v2.4; расширение поверхности до проверки С1 — второе, по правилу доски это красный, дальше — стоп на новые поверхности до живого пилота (`BOARD.md`, строка бюджета). Живые пилоты с командами — впереди; раздел 12 методологии перечисляет её собственные риски, `BOARD.md` держит их как ставки с тестами и порогами.
