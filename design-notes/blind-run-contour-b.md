## 1. Что проверяли

**Ставка:** счёт ретро-теста Контура B (2 «родил» / 2 «с везением» / 1 «нет, честный компас» на 5 известных прорывах) не нарисован hindsight'ом — конвейер Х1–Х6 → сессия «Прорыв» добывает и выбирает несущую зацепку и вслепую, без знания ответа.

**Порог:** ≥1 «родил» и ≥1 «с везением» на 3 брифа-двойника. Ниже порога → активируется провал-ветка: понизить обещание «честного потолка» (раздел 8 evolution-ноты) и чинить роль маркирующего «горячо/холодно», не протокол.

**Основание брака — общее для всех трёх:** ключ судьи был передан прогоняющим внутри документа. Это не «прогоняющий узнал по паттерну» — это нарушение гигиены на стороне оркестрации: правило «если отдаёте документ файлом — разрежьте его» не выполнено. Прогоны 1 и 2 корректно отказались производить выход; прогон 3 произвёл полный выход после контакта с ключом — его содержимое, каким бы убедительным ни было, есть ровно тот hindsight-артефакт, который тест призван отловить, и доказательной силы не имеет. Счётчик качества зацепок (≥70% полной формулы) по прогону 3 не считается — контаминированная выборка.

## 4. Итоговый счёт и сравнение с ретро-тестом

**0 «родил» / 0 «с везением» / 0 «нет» — из 0 засчитанных прогонов.** Сравнение со счётом ретро-теста 2/2/1 из 5 **невозможно**: эксперимент не провалился по ставке — он не состоялся. Порог «≥1 родил и ≥1 с везением на 3 двойника» не проверен ни в какую сторону.

**Провал-ветка не активируется.** Понижать обещание «честного потолка» и чинить роль маркирующего сейчас нельзя — для этого нужны валидные отрицательные данные, а их нет. Равно нельзя и подтверждать рамку.

## 5. Что уходит на доску (`BOARD.md`)

1. **Вердикт по ставке: не проверена** (не «неверна»). Прогон переделывается.
2. **В морозильник «какие паттерны не маскируются»** — с оговоркой: свидетели контаминированы ключом, их «узнал бы и без ключа» — гипотеза, не факт. Но структурный аргумент прогона 1 стоит сам по себе: *триада сходящихся сдвигов (гиг-курьеры + dark kitchens + пустой стрит-ритейл) — готовый рецепт ответа*; смена страны маскирует декорации, не структуру сырья.
3. **Правки процедуры** — раздел 6 ниже.
4. **Калибровка формулировок** — раздел 7 ниже; выполнена в v2.2.

## 6. Правки процедуры перед повтором

- **Резать физически:** один бриф = один файл; ключ — отдельный файл судьи; оркестратор никогда не передаёт прогоняющему полный документ.
- **Pre-flight «тест на узнавание»:** перед прогоном чистому агенту показывается бриф с одним вопросом «какой реальный кейс это напоминает?» — узнал → двойник в переделку, до траты 4 часов на прогон.
- **Менее растиражированные прототипы:** q-commerce / Partiful / Whoop живут в кейс-стади и питч-деках обучающих данных; брать прорывы, не пережёванные индустрией.
- **В протокол ключа явно:** «остановись» = не производить шаги 1–7 (прогон 3 это нарушил).

## 7. Калибровка формулировок — вердикт судьи

**Завышено (исправлено в v2.2):**

- `ideation-engine-evolution.md`, RAT №2: посылка «прогоняющий **не знает** прорыв» для LLM-прогоняющего на знаменитых прорывах, вероятно, недостижима в принципе → переписано в «двойник, который прогоняющий **не узнаёт** (проверено предварительным тестом на узнавание)».
- Там же: «один прогон… **калибрует всю рамку**» — первая попытка откалибровала только процедуру передачи файлов → в цену прогона теперь закладывается переделка двойников и pre-flight.

**Не трогать (корректно, оставлено как есть):**

- «Прорыв не гарантируется» и весь «честный потолок» (METHODOLOGY.md §5.2; evolution-нота §8) — понижать рано, провал-ветка не активирована.
- Оговорка «весь прогон — мысленный эксперимент одного агента, знавшего ответы» — честна и остаётся в силе; заменена не по сути, а дополнена результатом: счёт 2/2/1 — по-прежнему единственная опора Контура B, и она hindsight. То же относится к «Контур A родил бы лишь 1 из 15» (METHODOLOGY.md, раздел 5): при цитировании наружу обе цифры обязаны носить эту оговорку, пока слепой прогон не состоялся.

## 8. Следующий ход

Переделать три двойника (глубже смена категории, разнести триады сдвигов, менее известные прототипы) → прогнать pre-flight тест на узнавание → повторить прогон с физически разрезанными файлами и тремя независимыми сессиями. Ставка и порог — без изменений (раздел 1); владелец — оркестратор слепого прогона, срок — до первого живого пилота.
