## 9. Попытка №2 (03.07.2026): забракована на pre-flight — слепота маскировкой недостижима

### 9.1 Процедура

Правки раздела 6 выполнены: файлы разрезаны физически (один бриф = один файл, ключ судьи — отдельный файл оркестратора), четыре новых двойника вместо трёх (запас на один брак), маскировка усилена по урокам попытки №1 — сменены страна, категория и структура декораций. Перед прогоном — pre-flight тест на узнавание: чистому агенту показывается бриф с вопросом «какой реальный кейс это напоминает?»; по ключу судьи прогон стартует только на брифах, прошедших pre-flight, минимум 3 из 4.

### 9.2 Результат pre-flight: 0 брифов из 4

Контроль слепоты забраковал все четыре брифа; прогоны не запускались — по процедуре ключа прогон после провала pre-flight не стартует. Запас на один брак съеден четырёхкратно. Эксперимент завершён — прототипы можно называть:

| Бриф | Прототип | Вид прозрачности |
|---|---|---|
| 1 «Spiżarnia» | Faire | назван прямо + механизм (free returns непроданного + net-60 = перенос риска запасов на платформу) + дата брифа привязана к моменту взлёта Faire |
| 2 «CampoBase» | Vanta | не назван — механизм вскрыт через аналог: Agriplace («audit-ready досье из записей фермы — буквально их продукт») + опознан шаблон конструирования брифа |
| 3 «Sorum» | Riverside | не назван — несущий инсайт вскрыт на pre-flight («асинхронный разбор записей ценнее live») + аналоги той же механики: Skillest / Trala / Tonebase |
| 4 «VinLab» | Levels | не назван — сегмент и задача достаются из памяти заранее (Vinmetrica / Sentia — «ровно тот сегмент»); уровень «с везением» добываем без движка |

Правило строгости («сомнение — в сторону брака») применено во всех четырёх. Браки 2–4 — **новый тип прозрачности**: прототип не узнан, но целевое направление извлекаемо из памяти о структурно идентичных реальных кейсах. Прогон не отличил бы работу движка от вспоминания.

### 9.3 Счёт и порог

**0 «родил» / 0 «с везением» / 0 «нет» — из 0 засчитанных прогонов.** Сравнение со счётом ретро-теста 2/2/1 снова невозможно; порог «≥1 родил и ≥1 с везением на 3 засчитанных» не проверен ни в какую сторону. Провал-ветка RAT №2 **не активируется** — валидных отрицательных данных о движке нет. Порог не трогать.

### 9.4 Что это значит для рамки Контура B

**Подтверждено:** pre-flight как дешёвый детектор — четыре брака пойманы за минуты, а не после 4×4 ч прогонов; процедурная часть правок раздела 6 работает. Побочная находка попытки №1 «смена страны маскирует декорации, но не структуру сырья» подтверждена чистыми свидетелями и расширена: смена категории и декораций тоже не маскирует **механизм**.

**Опровергнуто:** «менее известные прототипы» как рецепт (вычеркнуто в разделах 6 и 8): Vanta / Riverside / Levels не названы — а суть всё равно прозрачна через смежные кейсы.

**Главный вывод:** посылка «прогоняющий не знает» для LLM-прогоняющего на реальных прорывах 2019–2024 из «вероятно, недостижима» стала эмпирикой: 4/4 брака при уже улучшенной маскировке, причём узнавание не требует называния прототипа. Две попытки — два разных фейл-мода (утечка ключа → прозрачность механизма): сходимость переделки двойников на реальных прототипах под вопросом.

### 9.5 Следующий ход — пересматривать инструмент, не только двойники

По букве ключа судьи — переделка двойников, порог не трогать. Но два «не состоялся» подряд с разными фейл-модами — показание пересмотреть инструмент; три варианта:

1. **Синтетический прототип:** оркестратор конструирует вымышленный «прорыв» с заданным несущим инсайтом, у которого нет опубликованного носителя механизма; проверяется тем же pre-flight («назови реальные аналоги» — аналоги уровня механизма = в переделку). Минус: теряется связь с «реально случившимся» — честно записать в ограничения.
2. **Судейство по цепочке вывода, а не по совпадению направления:** прогоняющий декларирует известные аналоги первой строкой; судится только выведенное из материалов брифа шагами ходов Х1–Х6 (детектор (г) переносится с «назвал прототип» на «направление появилось раньше, чем зацепка»). Дешевле, но требует правки ключа.
3. **Перенос веса RAT №2 на живой пилот:** признать, что дешёвая слепая калибровка LLM-агентом упирается в память модели; метрики — из RAT №1 на первых 2 живых проектах (доля полных зацепок ≥70%, «удивило» ≥1/3) — там будущее по-настоящему неизвестно всем.

**Честный итог:** движок по-прежнему не опровергнут и не подтверждён слепым прогоном; подтверждён — pre-flight как дешёвый детектор; опровергнута — достижимость слепоты маскировкой декораций на реальных прототипах.
